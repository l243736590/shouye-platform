# 售业 MVP 运营闭环清单

## 已完成到可演示

1. 充值订单：用户可提交消费积分充值申请，后台确认收款后入账。
2. 提现申请：用户可提交可提现积分结算申请，提交后先冻结积分，后台审核后处理。
3. 积分流水：悬赏锁定、悬赏退款、回答采纳、付费内容解锁、提现冻结、提现驳回、充值入账和退款扣回都有流水记录。
4. 悬赏争议：后台可处理退款、申诉和恶意采纳争议。
5. 商家线索：后台可查看、搜索、筛选、分配、备注和导出咨询线索。
6. 基础风控：发帖、提问、回答、举报、点赞、商家线索、提现和充值均有频率限制或重复提交控制。

## 上线前必须确认

1. 微信支付或其他支付通道完成商户开通、回调验签、订单对账和退款流程。
2. 可提现积分的结算比例、手续费、税务责任、最低起提和打款周期写入正式规则。
3. 支付、提现、商家服务、线下服务和虚拟积分相关条款由律师或合规顾问复核。
4. 微信分享卡片必须用真机转发测试，并处理微信缓存。
5. 小程序发布前需要在微信开发者工具里完成上传、体验版、提交审核和隐私接口配置。

## 真实运营后继续补强

1. 仲裁：补完整证据上传、客服备注历史、双方申诉记录和最终裁决记录。
2. 退款：补自动退回逻辑、部分退款、超时未采纳退款和支付通道退款。
3. 风控：补设备指纹、同设备多账号、异常点赞、刷收藏、批量注册和商家重复广告识别。
4. 商家：补商家账号、认证材料、服务上架、线索分配、订单记录和商家违规处理。
5. 数据：补运营看板、转化漏斗、内容审核效率、商家线索转化和搜索关键词统计。

## 演示时的安全说法

第一版先做“留学生问题解决社区 + 商家服务连接平台”的闭环验证。积分和提现在 MVP 中以人工审核为主，正式自动支付和提现需要等支付通道、风控、合规规则和小程序审核全部准备好后上线。
